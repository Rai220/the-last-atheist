# Последний Атеист — Центральная идея

## Правила генерации арта

**ВАЖНО: Никаких текстовых надписей на изображениях!** AI-генераторы (Gemini, Imagen, DALL-E) постоянно делают ошибки в тексте — кривые буквы, бессмысленные символы. Все промпты должны содержать `no text, no letters, no words, no signs, no labels, no writing on the image`. Любой текст в игре показывается через Monogatari UI, а не на фоновых изображениях.

**ВАЖНО: Не использовать SVG как замену сгенерированным изображениям.** Если Google-сервер для генерации изображений недоступен, не пытаться делать заглушки, мокапы или "картинки" через SVG. В таком случае лучше вообще ничего не генерировать, чем подменять результат SVG-файлом.


## Ключевая тема: Симметрия веры

Атеист, всю жизнь требовавший доказательств от верующих, оказывается в загробном мире — и приходит к единственному для себя приемлемому объяснению: **это симуляция**. Кто-то (возможно, он сам в будущем, или ИИ, или внешний наблюдатель) запустил программу, где сознание атеиста помещено в сеттинг ада и рая — чтобы потроллить, провести эксперимент, или просто поприкалываться.

Он **искренне верит** в эту гипотезу. Он пытается доказать её всем вокруг — демонам, душам, Богу. Но у него **нет ни единого доказательства**. Только внутренняя убеждённость. Только "я это чувствую, я это знаю".

И тут его накрывает осознание:

> Он оказался в **абсолютно симметричной ситуации** с теми верующими, которых высмеивал всю жизнь. Они верили без доказательств — и он теперь верит без доказательств. Они говорили "я просто знаю" — и он теперь говорит "я просто знаю". Все вокруг смотрят на него как на безумца — точно так же, как он смотрел на верующих.

## Скрытая отсылка: "Контакт" Карла Сагана

Прямая аллюзия на фильм/роман "Контакт":
- Элли Эрроуэй (учёный-атеист) переживает контакт с внеземной цивилизацией, но не может его доказать
- Палмер Джосс (богослов), который всю жизнь говорил о вере без доказательств, теперь требует от неё доказательств
- Роли меняются: учёный просит "просто поверьте мне", а верующий говорит "где факты?"

В нашей игре:
- Алексей всю жизнь требовал доказательств от верующих
- Теперь ОН верит (в симуляцию) без доказательств
- Бог/демоны/души требуют от НЕГО доказательств
- Он не может их предоставить — и понимает, каково это

## Как это меняет структуру игры

**Концовка "Матрица"** — скрытая четвёртая концовка, самая глубокая:
- Доступна при высоком `argument_quality` + высоком `wtf_level` (умный, но упрямый)
- Алексей формулирует "гипотезу симуляции" и пытается её доказать
- Проходит через серию сцен, где каждый его аргумент зеркально повторяет аргументы верующих
- Кульминация: момент осознания симметрии — тишина, пауза, и он ПОНИМАЕТ

**Зеркальные диалоги** (ключевой приём):
| Что говорили верующие на Земле | Что говорит Алексей в аду |
|---|---|
| "Я просто чувствую присутствие Бога" | "Я просто чувствую, что это симуляция" |
| "Посмотри вокруг — мир слишком совершенен для случайности" | "Посмотри вокруг — ад слишком абсурден для реальности" |
| "У тебя нет доказательств, что Бога НЕТ" | "У тебя нет доказательств, что это НЕ симуляция" |
| "Ты просто не хочешь видеть правду" | "Вы просто не хотите видеть паттерны" |
| "Однажды ты поймёшь" | "Однажды вы все поймёте" |

## Финальный твист

Бог (или голос) говорит Алексею:

> "Ты прав. Или не прав. Это не имеет значения. Потому что теперь ты знаешь, каково это — верить без доказательств. И теперь ты знаешь, почему они верили."

Экран гаснет. Ни подтверждения, ни опровержения. Игрок так и не узнает — симуляция это или нет.

## Пасхалки: мемы рунета (и не только)

**Принцип:** мемы должны быть ненавязчивыми — одна фраза, одно слово, деталь фона или внутренний монолог. Тот, кто знает — узнает и улыбнётся. Кто не знает — прочитает как обычный текст. Никаких «смотрите, это мем!» — только органика.

### Где размещать мемы
- **Внутренний монолог MC** — основное место для мемов; Алексей IT-шник, он думает мемами

- **Фоновые детали** — надписи, объявления, формы (описанные текстом)
- **НЕ в репликах Бога** — Бог не мемит
- **НЕ в серьёзных сценах** — breakdown, смерть, Лилит-предательство должны оставаться чистыми

## Cursor Cloud specific instructions

### Tech stack
- **Engine:** Monogatari visual novel engine (bundled in `engine/core/`)
- **Runtime:** Bun (dev server), vanilla HTML/CSS/JS (no build step for development)
- **Package manager:** Bun (`bun.lock`)

### Running the dev server
```
bun run serve
```
Starts a static file server with WebSocket live-reload on **port 5100**. No database or backend services are needed — the game is entirely client-side.

### Web build
```
bun build-web.ts
```
Outputs a production build to `./build/web` (strips debug script from HTML).

### Key caveats
- There is no ESLint or TypeScript config — the game JS files are plain vanilla JS. No lint command is available.
- There are no automated tests in the repository.
- The `package.json` `scripts.start` uses `yarn electron .` (Electron desktop wrapper) — this is **not** the primary dev workflow. Use `bun run serve` for browser-based development.
- Python scripts (`generate_art.py`, `generate_music.py`, `generate_scenes_pillow.py`) are offline asset-generation tools requiring `GEMINI_API_KEY`. They are not needed to run or develop the game.
- Game scripts live in `js/scripts/` (prologue, judgment, hell, endings) and `js/` (characters, assets, options, storage, effects, main, script).
